## EXECUTIVE SUMMARY

Round 5 audit focuses on **test infrastructure maturity and coverage depth** following r4's successful integration of multiplayer/audio tests. The suite is **functionally complete** (584 tests, all passing), but reveals **critical infrastructure gaps**:

1. **HIGH:** Coverage tool installed but **unconfigured** — `.coveragerc` missing, branch coverage disabled. CI/local visibility of actual coverage unknown.
2. **HIGH:** Three **critical engine paths have zero unit test coverage**: `allocache()` quick-path, `drawrooms()` render pipeline, `SE40_Draw()` status-list iteration.
3. **MEDIUM:** Hypothesis library installed but **severely underutilized** — only 4 tests use `@given`, massive opportunity on pure-function modules (palette, GRP packing, byte helpers).
4. **MEDIUM:** CI/local test drift: `--runslow` tests (30 total) **never invoked in CI**, creating blind spot for struct-size regression detection in CI.
5. **MEDIUM:** Test isolation good (tmp_path used correctly) but one hard-coded `captures/` path reference in playtest could leak between test runs.
6. **MEDIUM:** No mutation testing readiness — `mutmut` not installed, but pure-function modules (`palette.py`, `grp_format.py`, `art_format.py`) are ideal candidates.

**Status:** **INFRASTRUCTURE-CONSTRAINED GROWTH** — test count is healthy, but tooling and depth need investment.

## FINDING 1: COVERAGE TOOL INSTALLED BUT UNCONFIGURED [HIGH]

### 1.1 pytest-cov present, but no .coveragerc

**Evidence:**
```bash
$ python3 -m pip show pytest-cov
Name: pytest-cov
Version: 7.1.0

$ ls -la .coveragerc
ls: cannot access '.coveragerc': No such file or directory
```

**Impact:**
- Coverage reports cannot be generated with `pytest --cov=...` (uses pytest-cov defaults)
- **No branch coverage** — only line coverage possible, missing decision paths
- **No exclusion patterns** — test fixtures and autogenerated code count toward coverage
- **CI/local parity unknown** — no reproducible coverage config

### 1.2 No coverage gate in CI

**File:** `.github/workflows/build.yml:38`

Current test invocation:
```yaml
- name: Run tests
  run: python3 -m pytest tests/ -v --tb=short
```

Does NOT include `--cov=...` or coverage reporting.

### 1.3 Impact Assessment

- **developers cannot measure coverage** locally without manual setup
- **merges happen with unknown coverage**, risking untested code paths
- **regressions in critical paths** (allocache, drawrooms) would not be detected

### 1.4 Recommendation

Create `.coveragerc` with:
```ini
[run]
branch = True
omit = tests/*, tools/generate_ai_voices.py, compat/*, SRC/*
[report]
exclude_lines = pragma: no cover, @abstractmethod, raise NotImplementedError
[html]
directory = htmlcov
```

Add to CI:
```yaml
- name: Run tests with coverage
  run: python3 -m pytest tests/ --cov=tools --cov-report=term-missing --cov-report=html
- name: Check coverage threshold (80% minimum)
  run: python3 -m pytest tests/ --cov=tools --cov-fail-under=80
```

## FINDING 2: CRITICAL ENGINE PATHS HAVE ZERO UNIT COVERAGE [HIGH]

### 2.1 Untested paths in SRC/CACHE1D.C and SRC/ENGINE.C

**allocache() quick-path** — Memory cache allocation
- File: `SRC/CACHE1D.C` (21 KB, 3 functions only)
- Function: allocache() has two codepaths:
  1. **Quick path**: `cachecount <= sizecache` (immediate reuse)
  2. **Suck pass**: invalidate old entries, alloc new
- **Status:** ZERO tests. No unit test validates either path.
- **Risk:** Memory corruption if cache eviction order is wrong

**drawrooms() render pipeline** — Sector-based BSP traversal
- File: `SRC/ENGINE.C` 
- Function: Renders visible sectors from player viewpoint
- **Status:** ZERO direct tests. Only indirect validation via playtest frame analysis.
- **Risk:** Polygon clipping errors, depth sorting bugs would not be caught until runtime.

**SE40_Draw() status-list iteration** — Effector rendering
- File: `SRC/ENGINE.C`, Effector SE40 (power-up)
- Function: Iterates linked list of active effectors, draws each
- **Status:** ZERO unit tests. Marked "safe" in r4 but no regression test added.
- **Risk:** Linked-list corruption would crash mid-iteration.

## FINDING 4: CI/LOCAL TEST DRIFT — SLOW TESTS NEVER RUN IN CI [MEDIUM]

### 4.1 --runslow skipped in CI

**File:** `.github/workflows/build.yml:38`

Current CI invocation:
```yaml
- name: Run tests
  run: python3 -m pytest tests/ -v --tb=short
```

**Impact:**
- 30 slow tests (`@pytest.mark.slow`) are **SKIPPED in every CI build**
- Struct-size compilation tests (2 tests) never validated in CI
- Audio generation subprocess tests (11 tests) never validated in CI
- Multiplayer CRC stress tests (3 tests) never validated in CI
- Hypothesis stress tests (4 tests) never validated in CI

### 4.2 Evidence

```bash
$ pytest -q                      # Local without --runslow
553 passed, 31 skipped (18.04s)

$ pytest -q --runslow            # Local with --runslow
# Would show: 583 passed, 1 skipped (~56s)
```

### 4.3 Recommendation

**Option A (simplest):**
```yaml
- name: Run all tests (fast + slow)
  run: python3 -m pytest tests/ -v --tb=short --runslow
```
Adds ~37s to build time; catches regressions in struct/CRC/audio paths.

**Option B (separated, clearer):**
```yaml
- name: Run fast tests
  run: python3 -m pytest tests/ -v --tb=short -m "not slow"

- name: Run slow tests  
  run: python3 -m pytest tests/ -v --tb=short -m slow
```
Same time overhead but clear separation; slow tests fail independently.

**Recommendation:**
```ini

    playtest: Visual playtesting — launches game headless and validates captured frames

    integration: Multi-component integration tests
    fuzz: Hypothesis property-based tests with generated inputs
addopts = -ra
filterwarnings =
    ignore::DeprecationWarning:pydantic
```
